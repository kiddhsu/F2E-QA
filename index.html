Q:宣告變數的方法、型別有哪些?

  • 方法
  • var 宣告一個可隨意更改其內容的變數
  • let 宣告一個可隨意更改其內容的區塊區域變數
  • const 宣告一個只可讀取的不可變常數
  • 型別(可用 typeof 來檢測)
  • number 數字，例如：12345
  • string 字串，例如：'Hello World'
  • boolean 布林，例如：true、false
  • null
  • undefined
  • object 物件，例如：{ name: 'Jack' }

Q:何為let/const跟var的差別為何?

  • 作用域：
  • let 、 const 的作用域為區塊作用域
  • var 的作用域為函式作用域
  • 重複宣告：
  • let 、 const 不能重複宣告
  • var 可以重複宣告
  • 重新賦予原始型別的值：
  • const 不能重新賦予原始型別的值
  • var 、let 可以重新賦予原始型別的值

Q:何為scope/closure/hoisting/prototype/DOM?

  • Scope 作用域意思就是，變數在程式中可以被存取的範圍
  • Closure 是指變數的生命週期只存在於該 Function 中，一旦離開了 Function，該變數就會被回收而不可再利用，且必須在 Function 內事先宣告
  • Hoisting 是一種把宣告提升到其所在區域內頂端的行為，意即程式會將 Function 中全部需要宣告的 Local Variable，提升到 Function 的第一行來執行，但不包含初始值設定
  • DOM 把一份 HTML 文件內的各個標籤，包括文字、圖片等等都定義成物件，而這些物件最終會形成一個樹狀結構(DOM tree)

Q:描述下列之間的不同script, script async & script defer?

  • script 整個網頁繪製會停下，等 js 下載完並執行完，網頁繪製才繼續
  • defer(延遲) 網頁繪製不會停下， js 在背景下載，待 DOMContentLoaded 再執行js
  • async(非同步) 網頁繪製不會停下， js 在背景下載;等js 下載完畢，網頁繪製停下，執行 js ;等 js 執行完畢，網頁繪製繼續

Q:為什麼把 CSS link 放在 head 之間，與將 JS script 放在 body 之前是個較好的主意？有什麼例外情形嗎？

  • CSS
  • link 放在 head 之間可讓頁面逐步呈現，放底部可能讓頁面空白
  • JS
  • 對於呼叫執行的，我們最好放在head裡，直接宣告或者引用外部檔案（head）
  • 對於即時執行的，我們在需要js執行的地方宣告或者引用外部檔案（body）
  • 建議在body的底部引入，保證html完成
  • 例外情形
  • 當腳本包含document.write(),可使用defer属性放在head中

Q:PWA是什麼

  • 徹底改善用戶體驗
  • Progressive 漸進式，提供每一位用戶做基本的瀏覽，若支援環境、可以提供更強大的功能
  • Responsive 響應式的用戶介面
  • Connectivity independent 不依賴網路連接，透過 service workers 可以在低頻寬甚至是離線的環境下瀏覽網站
  • App-like 讓網站可以具有像 APP 般的瀏覽速度等優點，提供更佳的用戶體驗
  • Fresh 藉由 service worker 自動更新網站內容
  • Safe 通過 HTTPS 來提供服務，確保資料安全性
  • Discoverable 像是推播通知等特性，吸引使用者注意、主動和使用者互動，提升用戶回流率
  • Installable 可以藉由 Add To Home，如同 App，會新增一個 icon，可以直接將網站加到手機桌面上做切換使用，不需要再透過 App Store 下載安裝
  • Linkable 可以將網站、透過 URL 相互傳遞分享

Q:pt/px/em/rem差別

  • pt單位名稱為點（Point），絕對長度單位，常用於印刷行業
  • px單位名稱為像素，相對長度單位，像素px是相對於顯示器屏幕解析度而言的
  • em單位名稱為相對長度單位。沒有一個固定數值，相對於當前對象內文本的字體尺寸。如當前對行內文本的字體尺寸未被人為設置，則相對於瀏覽器的預設字體尺寸
  • rem是CSS3新增的一個相對單位（root em，根em）， rem是相對單位，是相對HTML根元素。

Q:null、undefined和 undeclared變數之間有什麼差異？你如何檢查？

  • null 等於變數是「沒有」的值，本身等於有值了，這個值就叫「沒有」
  • undefined 宣告但尚未給予值
  • undeclared 是還沒有經過宣告的變數

Q:什麼是同步非同步

  • 非同步就是每個任務各做各的，不用等其他任務完成，再進行下一個任務

Q:git-flow/gitlab-flow

  • Git-flow
  • 優點 清晰可控
  • 缺點 較為複雜，需要同時維護 master 分支與 delevop 分支兩個長期分支
  • gitlab-flow
  • Git flow 和 Github flow 的综合，只認存在一個主要分支 Master 分支，他是其他所有分支的「上游」
  • 「版本發佈」- 每當有一個穩定的版本，就要從 master 分支拉出一個分支。只有修補 Bug 時，才允許將代碼合併到這些分支
